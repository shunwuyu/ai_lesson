# VS Code Markdown 微信公众号预览插件 - 需求分析文档

## 1. 项目概述

开发一款 Visual Studio Code 扩展插件，为 Markdown 文件提供微信公众号样式的实时预览功能。

## 2. 核心需求

### 2.1 高优先级需求

#### 2.1.1 预览窗口激活

**需求描述**：当用户在 VS Code 中打开 Markdown 文件时，提供便捷的方式打开预览窗口。

**用户场景**：
- 用户打开任意 Markdown 文件（.md 或 .markdown 扩展名）
- 用户希望查看该文件的微信公众号渲染效果
- 用户通过点击按钮的方式打开预览窗口

**期望行为**：
- 插件检测当前活动编辑器中的 Markdown 文件
- 提供一个按钮（位置待定）供用户点击
- 点击按钮后打开独立的预览面板/窗口

---

#### 2.1.2 Markdown 内容渲染

**需求描述**：预览窗口使用 md-wx 组件对当前 Markdown 文件的内容进行渲染。

**用户场景**：
- 用户查看 Markdown 文件的渲染效果
- 渲染效果应符合微信公众号文章的排版样式

**期望行为**：
- 预览窗口读取当前 Markdown 文件的完整内容
- 使用 md-wx 组件进行渲染
- 渲染结果呈现微信公众号样式的排版和格式

---

#### 2.1.3 实时预览同步

**需求描述**：当 Markdown 文件内容发生变化时，预览窗口的内容应自动更新，保持同步。

**用户场景**：
- 用户在编辑 Markdown 文件
- 用户希望立即看到编辑后的渲染效果

**期望行为**：
- 监听 Markdown 文件内容的变更
- 文件变更后自动触发预览窗口内容更新
- 更新过程平滑，不影响用户编辑体验

---

#### 2.1.4 一键复制功能

**需求描述**：预览窗口提供一键复制功能，将渲染后的内容复制到剪贴板。

**用户场景**：
- 用户对预览效果满意后
- 用户需要将内容复制到微信公众号编辑器或其他地方

**期望行为**：
- 预览窗口中提供复制按钮或入口
- 点击复制后，将渲染后的完整内容复制到剪贴板
- 复制后的内容格式适合直接粘贴到微信公众号编辑器

### 2.2 中优先级需求

#### 2.2.1 图片粘贴功能

**需求描述**：当用户在 Markdown 编辑器中粘贴图片时，将图片以链接形式插入到粘贴位置。

**用户场景**：
- 用户从剪贴板粘贴图片
- 希望图片以外部链接的方式嵌入到 Markdown 中

**期望行为**：
- 检测到剪贴板内容为图片
- 将图片上传至配置的存储服务
- 将图片的外部访问链接以 Markdown 图片语法 `![描述](链接)` 的形式插入到当前光标位置
- 支持自定义图片描述

**可选配置**：
- 支持配置不同的图片存储服务
- 支持配置图片上传的参数（如命名规则、路径等）

---

## 3. 非功能性需求

### 3.1 用户体验

- 预览窗口响应速度快
- 操作流畅，无明显卡顿
- 界面简洁，易于理解和使用

### 3.2 兼容性

- 兼容主流版本的 Visual Studio Code
- 兼容常见的 Markdown 语法

### 3.3 稳定性

- 在预览过程中不影响编辑器的正常使用
- 处理异常情况时有合理的错误提示

---

## 4. 需求优先级总结

| 需求 | 优先级 | 说明 |
|------|--------|------|
| 预览窗口激活 | 高 | 核心功能入口 |
| Markdown 内容渲染 | 高 | 核心渲染能力 |
| 实时预览同步 | 高 | 核心交互体验 |
| 一键复制功能 | 高 | 核心输出能力 |
| 图片粘贴功能 | 中 | 增强型功能 |

---
