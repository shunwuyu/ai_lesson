脑海里突然蹦出一个特别的产品想法，变成一份能够交由开发团队的PRD（产品需求文档） 文档。

传统需要花好几天的时间， 反复沟通， 梳理流程，考虑各种异常情况。

## 24小时随时待命的产品经理

只需要给它一个模糊的想法，他会帮你澄清需求理清逻辑， 最后输出一份完整的PRD文档。

## 痛点

- 写一份完整的PRD 文档 ， 往往需要花费1-2天时间。
PRD 本质上并不是在写文档， 而是一个梳理过程。

把模糊的业务想法， 拆解成具体的用户故事。

梳理核心业务流程和状态集。

考虑各种异常链接情况。定义清晰的验收标准。

很多产品想法在早期阶段，往往是非常模糊的。

比如， 我想做一个积分兑换功能， 但是积分怎么获取怎么消费，有没有有效期，需不需要会员等级等。这些问题没有想清楚， PRD 文档就没有办法继续往下编写。 

## PRD skill

在你只有一个模糊想法的时候， 它会主动向你提问。自动生成结构完整， 逻辑严密的PRD文档。

有哪些作用呢？

首先， 他能节省大量的时间。
从最初的两到3天时间， 缩短到5分钟。

只需要回答几个问题，剩下的全部都能实现自动生成， 质量有保证。

快速生成多个版本的PRD文档， 然后进行对比。从中选择最优秀的方案。

## PRD Agent Skill 是如何实现的

三层架构设计
- 核心哲学层
优秀的PRD=明确的出发点+产品的逻辑图+变态的边界考虑-任何多余的废话
这个skill的灵魂， 定义了什么是"好的PRD"  简洁、逻辑严密、考虑周全。

- 工作模式层
多轮迭代澄清（Refine-Before-Write）
1. 需求诊断 -> 2. 逻辑构建 -> 3. 细节补全
PRD skill 不会看到需求就开始文档编写。而是先评估需求的模糊程度，
如果信息不足，它会主动使用Ask User Question 工具，向用户发出提问，来补充相应的
背景信息。
- 输出标准层
8 大标准模块， 涵盖PRD的完整要素
业务背景 | 术语定义 | 用户故事 | 功能清单
业务流程 | 状态机 | 边界场景 | 数据埋点
