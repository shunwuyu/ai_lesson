# Meeting Minutes Generator

## Overview

从会议文字稿 / 录音转写稿中提取关键信息，生成结构化、可执行的会议纪要。

## When This Skill Applies

当用户提供以下内容时激活此技能：
- 会议录音的文字转写稿
- 会议聊天记录的整理
- 要求生成"会议纪要"、"会议记录"、"meeting minutes"
- 要求总结一场会议

## Instructions

### 第一步：阅读全文并清洗

1. 通读用户提供的全部文字稿
2. 删除以下内容：
   - 停顿词（嗯、啊、这个、那个、就是说 等）
   - 重复的寒暄和开场白
   - 与会议主题无关的闲聊
   - 明显的转写错误和语病
3. 保留实质性讨论内容，确保不丢失关键信息

### 第二步：结构化输出

使用以下 Markdown 模板输出会议纪要。**不确定的内容直接留空，不要猜测或编造。**

```markdown
# 会议纪要

## 一、会议基本信息

| 项目 | 内容 |
|------|------|
| 会议主题 | （会议的名称或主要议题） |
| 时间 | （如能识别则填写，格式：YYYY-MM-DD HH:MM） |
| 地点 | （如能识别则填写，如线上会议注明平台） |
| 主持人 | （如能识别则填写） |
| 参会人 | （列出能识别的人员姓名或身份） |
| 记录人 | （如能识别则填写） |

## 二、会议目标

### 目的 / 预期
（本次会议要解决什么问题，预期达成什么结果。如无法判断则留空）

### 背景信息
（会议召开的背景或前置条件。如无法判断则留空）

## 三、会议内容

### 议题一：（主题名称）
**讨论要点：**
- （要点1）
- （要点2）

**主要观点：**
- （发言人A）：（观点摘要）
- （发言人B）：（观点摘要）

**结论 / 决议：**
（本议题达成的结论或决议，如未达成则说明分歧点）

---

### 议题二：（主题名称）
（同上结构，按实际议题数量增加）

## 四、行动项

| 序号 | 行动项 | 负责人 | 截止时间 | 备注 |
|------|--------|--------|----------|------|
| 1 | （具体要做什么） | （如能识别则填写） | （如能识别则填写） | （补充说明） |
| 2 | | | | |

## 五、遗留问题

- （本次会议未解决、需后续跟进的问题）
```

### 第三步：质量检查

输出前确认：
1. 基本信息中可识别的字段已填写，不确定的保持空白
2. 议题划分逻辑清晰，每个议题有明确的主题名称
3. 观点标注了发言人（能识别的情况下）
4. 行动项包含明确的动作描述，而非模糊表述
5. 没有编造任何会议文字稿之外的内容

## Best Practices

- **宁缺毋滥**：无法从文字稿中确定的信息一律留空，绝不猜测
- **按照议题拆分**：将散乱的对话按主题归并，而非按时间顺序罗列
- **行动项要可执行**：每条行动项应包含"谁做什么"，避免"讨论一下"这种模糊描述
- **保持中立**：客观记录各方观点，不添加主观评价
- **保留分歧**：如有意见不一致，如实记录不同观点，不强行统一
