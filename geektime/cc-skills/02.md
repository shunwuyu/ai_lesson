# 从一张截图到完整产品分析

我们来玩一个更刺激的，通过一张产品截图，让Ai 自动帮你生成一份专业的竞品分析报告。

你只需要上传一张截图， 泡杯咖啡回来， 一份3000多字的竞品分析报告就静静的躺在那里。

这是大家都可以掌握的技能。 

## 产品逆向分析skill 

- 刚接手一个新项目， 要求在一周之内完成竞品分析报告。
- 打开竞品分析APP, 首页刷到详情页。从注册流程用到支付环节，边用边截图
  边记录要点。 忙活了4、5个小时， 终于整理了自己觉得还完整的分析报告。

一发给老板， 老板灵魂夺命三问
- 这个产品的目标用户到底是谁？
- 为什么竞品需要这么设计
- 背后的业务逻辑是什么？
- 这个功能需求模块的优先级是怎么确定的？
  是P0（致命紧急）还是P1（高优重要）

  懵了， 刚刚记录的全是是什么， 按钮放在哪里， 列表长啥样，页面之间的跳转逻辑是什么样子的。

  对于为什么需要这么做， 这背后究竟意味着什么，完全没有更深入的洞察。

其实， 这不是能力的问题， 纯靠手工竞品分析本身就是一件极其低效的事情。
传统方式主要靠个人经验积累， 同一个产品， 老产品经理一眼看透底层逻辑。

新人只能看到表面功能。

好消息， 新人只要使用这个skill, 不但可以将工作效率提升4-8倍。 产出的分析报告达到专业水准。

## 案例

大家都参与过 点转盘、集卡片， 领现金这些游戏活动。
看着简单，背后全是产品、心理、算法、变现的层层设计。
jp/1.png

```
一、产品定位
类型：现金激励 + 转盘抽奖 + 集卡游戏化营销工具
目标：DAU 提升、广告变现、社交裂变、支付信息沉淀
用户：下沉市场、中老年、时间多、对小额现金高度敏感
二、核心增长套路
临门一脚错觉：金额接近可提现（39.99→40 元），真实门槛藏在「集小马 / 集卡」
概率阶梯收割：前期高概率掉卡建立信心，后期极低概率逼用户看广告 / 分享
强心理操控：沉没成本、损失厌恶、倒计时焦虑、FOMO 错失恐惧
提现层层设卡：首提低门槛（40 元）→后续提额→强制实名 + 绑卡
三、核心模块作用
进度卡片：制造紧迫感，放大金额与落差
抽奖转盘：前端娱乐化，后端概率控制
CTA 按钮：强化 “易提现”，引导任务
助力入口：低成本社交流量裂变
四、变现闭环
用户抽奖 → 次数不足 → 看广告 / 分享 / 拉新 → 平台广告收入 + 新增用户 + 支付信息 → 提高留存与商业价值
```

左手拿到社交裂变流量， 右手沉淀了用户的实名支付信息，完成整个商业闭环。

## SKILL 初步理解
一个完整的Agent Skill 由skill.md 、scripts、references、assets 组成。

skill.md 是核心
  - 三部分， 头部、中部、尾部
  - 头部
    我是谁（name）， 什么时候调用我(description)
  - 中部
    五部产品逆向分析法
    宏观定位 这个产品是干什么的， 服务于谁， 解决什么问题
    、微观拆解 （视觉、交互、 信息架构三个方面进行分析， 任何UI 设计都是由这三个方面组合）
    、逻辑推演、（为什么）
    流量可视化、价值评估
scipts node/python 脚本
references 存放文件
assets 存放模版文件
