# 产品原型设计skill（Product Prototype Design）

作为产品经理或开发者， PRD写了300多行。
但是每当跟AI沟通的时候， 他觉得你的布局设计不够合理。
和开发沟通的时候， 它理解不了你的交互逻辑。

这种手动翻译需求为产品原型设计的时代过去了，
产品原型skill, 可以让你直接跳过画图阶段。
将PRD文档， 转成可部署高保真的原型



## 电商积分兑换案例
![](1.png)
一、行业基因的自动对齐
大多数AI模型生成的界面设计中， 往往都会有一股浓烈的AI味。
具体表现为， 生成结果中大量采用蓝紫渐变配色和过渡使用的圆角设计。

产品原型设计skill, 采用了行业驱动设计的概念。
会自动根据用户的需求判断所处行业

比如是电商行业， 会自动采用红橙色系的配色方案，并且生成促进转化的页面布局结构。
第二 采用了演示胜过表述的理念
他会主动根据需求描述的功能， 生成对应的逻辑代码。
当文档中写了， 点击扣除积分按钮，并实时更新状态。他能实时生成一个
能实时更新state的原型设计，实现模拟积分加减， 库存预警， 状态切换功能

第三步 这个skill 实现了部署零门槛

做完原型设计后，怎么分享给团队成员观看。 
skill 集成了部署逻辑，生成完成后会自动进入发布流程。

## 执行流程
![](2.png)

需求识别、行业推断、规范匹配、原型构建和交付与迭代5个阶段。

### 阶段一 需求识别
skill 会根据输入内容，进行意图识别， 判断提供的信息是否充分。
确保进入设计前， 需求定义是清晰完整的。

### 阶段二 行业推断
Skill 会根据提供的PRD文档中提取关键词，然后映射到具体的行业分类
比如说当提取到外卖， 会是餐饮行业。skill 内置了科技、金融、电商
、医疗、教育等十余个行业
不同的行业对视觉语言的要求是完全不同的。
### 阶段三 视觉设计规范加载
关键决策环节
skill 中包含行业设计规范文件，包含四类核心参数，
分别是 配色方案、字体排版、容器与边角样式，交互动效

确保输出的设计具备行业一致性， 而非千篇一律的生成式风格。

### 阶段四 原型构建

html5, tailwindcss, vanilla js 并且通过cdn 引入。

### 阶段五 一键部署

我们有一份完整的积分兑换功能PRD 文档， 包含用户故事、功能清单、业务流程、
状态集、边界场景等内容。
