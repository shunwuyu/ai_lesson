- 大文件上传

大文件上传通过分片实现断点续传，上传前对文件计算 hash，实现秒传与断点命中。每片支持并发上传，失败可重试，最终合并。前端用 File.slice 拆分，结合 web worker 做 hash，后端存储片信息与状态，支持分片合并与断点校验，提升稳定性与用户体验。

- 介绍大文件上传何为临时存储，何时删除？
临时存储是指服务端在接收大文件的各个分片时，先将这些分片暂存到磁盘或缓存中，直到所有分片上传完成，再进行合并生成最终文件。
{fileHash}_{chunkIndex}

何时删除临时存储：
    - 上传成功后合并完成
    - 超时未完成上传：
    - 用户取消或主动清除：

- 断点续传，客户端存储哪些东西

    - 文件唯一标识（hash） webworker 计算
    通常是整个文件的 MD5 或 SHA-1 哈希，用于判断文件是否上传过、是否秒传。
    - 分片信息（chunk list）
    每个分片的编号、大小、总数（如第几块、共几块），用于恢复上传进度。
    - 已上传分片列表
    用于从断点处继续，只上传未完成的分片。可由后端返回，也可本地记录。
    - 上传任务状态（可选）
    如上传开始时间、是否暂停、失败重试次数等，方便状态管理。
    - 文件名 + 大小 + 类型（可选）
    辅助用于展示进度或验证是否是同一个文件。
