# 代理模式

代理模式（Proxy Pattern）是设计模式中的一种结构型设计模式（Structural Design Pattern）

它的核心思想是：

通过一个“代理对象”来控制对真实对象的访问。

## 为什么需要代理模式

现实生活中的例子：

明星 ← 经纪人 ← 客户
房东 ← 中介 ← 租客
工厂 ← 代理商 ← 用户

用户并不直接接触真实对象，而是通过代理完成操作。

软件开发中也一样：

有时候我们不希望用户直接访问某个对象，而是：

- 访问前做权限校验 没有就不让访问
- 做缓存 走cache
- 记录日志 


二、代理模式结构

代理模式通常包含三个角色：

Client（客户端）
Proxy（代理对象）
RealSubject（真实对象）
