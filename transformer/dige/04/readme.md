# self-attention 怎么计算

- 输入经过编码后(tokenizer->embedding)得到两个向量(初始化)
  x1（thinking）, x2(mechines)  代表两个词元

- 得到当前词语上下文的关系， 可以当做是加权。
  我们需要知道对于第一个词看来，thinking, mechines 两个词上下文 分别对第一个词有什么贡献。

  对二个词看来， thinking, mechines 两个词上下文 分别对它有什么贡献。

  然后做重新的向量编码。

  Transformer（论文， 8个作者）， 为了使这件事情（计算序列内各词元间相互依赖的注意力权重）做得更好。神经网络无非就是提取特征。

- 构建三个矩阵分别来查询当前词和其他词的关系， 以及特征向量的表达。

  找3个矩阵， 分别是Queries, Keys, Values。 Q,K,V
  以W 命名， 权重 Wq, Wk, Wv

  借助三个辅助矩阵， 通过 QKV 映射将原词向量转为查询、键、值特征向量

  x1, x2, 两个向量， 还有个Qq 矩阵  通过矩阵算法（乘法）后得到 特征向量q1, q2 
  其他的也是一样  由我们的x1, x2输入， 经过三个矩阵后，得到三种不同的特征向量。
  Query 要去查询的， 有什么关系。q1, q2 你手里的寻人需求，拿着标准去匹配他人 打篮球阳光男孩 还浪漫 上海985高校的
  Key 等着被查的， k1, k2  简历 学校， 爱好 等
  单词之间的关系都查到了， 关系大的 当前这个词的特征多点往里输入。
  关系少的， 少点往里输入。
  Value 实际的全部特征  v1, v2, 就是实际的特征信息了

## 计算
对于每个向量都有 xn, qn, kn, vn   

都有实际的值， 

怎么表示第一个词和每个词的关系？ 
q1*k1 , q1*k2 ..... 
都是Q1, k1, k2, k3, ... 这些词之间的关系。
q1和每个k 之间的关系？ q 与k的内积（两个矩阵所有对应位置元素相乘，再全部加起来）有多匹配

用坐标系来理解， x1 和x2 无关， 其内积必为0， 夹脚垂直。
什么是有关系？ 离得越近，越有关系。 
相关性越大， 内积也越大。 
内积越大， 相关性越大。
相关性越大， 等下做当前第一个词编码的时候， 会往里输入更多的特征。

这样我们就得到了每个词和其他词的相关程度。 而且不管句子多长， 都能算出来。
