https://leetcode.cn/problems/3sum/

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。

- 暴力破解 时间复杂度 O(n^3) 空间复杂度 O(1)
- 找三个不同下标的数使和为0，且返回不重复的三元组
  - 先对数组排序，遍历第一个数（遇到相同数跳过以去重） 
  排序让数组有序，便于使用双指针快速逼近目标，并能轻松跳过重复元素，从而高效去重、降低复杂度。
  - 固定一个数，化为两数之和
  先选定其中一个数不动，把三数之和为 0 的问题，变成“在剩下的数里找两个数，它们的和等于这个固定数的相反数”。
  - 双指针逼近目标
  用两个指针分别从数组的两端向中间移动，根据当前三数之和与目标的大小关系，调整指针位置。
  由于数组有序，当和大于目标时，右指针左移；当和小于目标时，左指针右移。
  当两个指针相遇时，说明当前固定数下不存在符合条件的两数之和，继续遍历下一个固定数。

  1.js 

  时间复杂度

  排序：O(n log n)  快排
  遍历每个 i（外层循环）：n 次
  每次 i 下做双指针：left、right 只会从两端向中间移动一次，所以是 O(n)

  外层 n × 内层 n = O(n²)
  排序 O(n log n) 被 O(n²) 覆盖，所以最终复杂度是：
  O(n²)
