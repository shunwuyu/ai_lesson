- 原则
  - 模块化切分（Divide and Conquer）
  不要试图一口吃成个胖子。AI 的上下文记忆和逻辑推理是有“带宽限制”（max_tokens）的。如果你一次性要求它“做一个完整的进销存ERP系统”，它大概率会写出一堆半成品。

  • ❌ 错误示范：“帮我做一个管理系统，要有登录、入库、鉴定、销售、财务报表，还要能发微信通知。”

  - ✅ 高效做法：将大需求拆解为小模块，一个一个来。

  • Prompt 1：“先帮我写一个‘登录页面’，只需包含账号密码输入框和登录按钮，UI参考极简风格。”

  • Prompt 2：“基于刚才的登录页，现在增加一个‘入库表单’，字段包含：品牌、成色、附件情况、回收价。”

  💡 背后的 WHY：AI 写代码就像人写文章，篇幅越长，逻辑越容易断层（Hallucination）。模块化可以让 AI 将全部算力集中解决当前这一个小问题，准确率会从 60% 飙升到 95%。
