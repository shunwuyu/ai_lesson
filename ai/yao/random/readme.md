# 大模型是怎么随机说话的？

今天来聊两个控制大模型随机的关键参数：

temperature 温度 | Top  K

## 大模型怎么说话的？

我们和大模型说 你好

llm 想的是怎么去接下一个词。

这个词怎么来呢？

llm 有很大的一个词表， 比如说

呀， 吗， 我， 的....

然后这个词表里每个词 都有被选中的概率， 

呀 0.1， 吗 0.6， 我 0.2， 的 0.1....

吗的概率是最大的， 但是llm 也不一定每次去选概率最大的那个。

因为这样的话呢， 回答都是一样的。 

可以根据我设的参数， 回答这个随机性， 是更保守点， 还是更
奔放一点？ 

## Temperature

主要是控制模型的发散程度， 温度低模型就会更谨慎， 更会选靠谱的
词。 

比如 temperature 0.2 , 选吗的概率是最大的。

这种确实很稳， 很有逻辑性， 但是稍显无聊， 有点单调。

把温度调到1.0， 那这个时候， 模型更发散， 更天马行空， 
连些更低概率的词都会选， 

你好美

你好坏
等这样低概率的词

这种创意多， 但可能会跑题， 所以你可以这么简单去记， 

想保守稳定 调低温度
想创意发散 调高温度

## Top K
- 作用是什么呢？
去限定你的候选的范围

他就好像一个筛子， 把所有可能的词， 按从低到高去做一个排序。

吗 0.6，我 0.2， 呀 0.1， 的 0.1....  坏 0.09

我们的k 选4， 那么 选吗， 我， 呀， 的

这些就是我们的候选词 candidates

最后就从里面随机去挑一个， 
比如选择 0.2 我  （随机）

所以你的 Top-K 设的越小， 它的输出就越集中， 越稳定，

设的越大， 它的输出就越有创意， 答案越花。 

Top K, 控制我要从多少个靠谱的词里面去挑，

然后温度呢， 我在挑的时候， 要激进呢？ 还是保守一点。

## 怎么使用

- 写代码， 偏向更准确事情的时候，
temperature: 0.2 低 
输出更保守、更具有稳定性， 更具有可信性。 
topK 可以选择默认值 8 
- 写文章， 或者小说的时候
温度调高一点， 0.8， 天马行空，
但是也不能跑题啊， 那么可以把topK 调小一点， 比如4
就会从一个更有逻辑性的上下文里去调。 

注意：

Temperature, TopK 不能同时调太高， 模型可能就疯掉了。
我们通过这两个参数掌握创意和精准度。
