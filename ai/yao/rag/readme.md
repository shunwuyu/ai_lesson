刚刚犯了两个问题：

怎么回答的更准、更有出处、更加可信呢？
RAG。
## RAG 本质， 拆开
G Generation 大模型的原始能力。
我们去问llm 问题， 大模型基于它之前训练的数据，得到了知识， 给我们生成答案。
这些回答，完全是基于模型的参数和它的训练数据，它不一定能告诉你信息的来源，
而且训练的数据是有个期限的， gpt 3.5 停留在2021年。它可能会编造一套看起来
很真实的内容。
rag 怎么改进呢？
让大模型再生成之前，先去检索资料或知识库， 然后告诉我们答案。 Retrieval
这个资料或知识库，可能是文档的集合，企业内部的文件，也可以是个API, 甚至是联网搜索的结果。

不是凭空生成回答， 而是先去检索，找到可靠信息，我在基于可靠信息（Prompt Augmentation）去给你回答。

就好像回答问题之前，你先去看了百科全书，检索完以后， 再去回答这个问题。

为什么RAG 解决了大模型硬伤呢？
它主要分成两个方面：
1. 它的信息可更新。 当世界发生改变的时候， 比如， 不需要再次训练模型， 只需要更新数据库，
就可以查到最新的信息。
2. 模型回答的内容是有出处的。
模型的回答是建立在检索基础上的， 可以把检索的内容（来源）和回答一并去输出。
看来就会好很多，大大降低了幻觉的风险。

最终的prompt 包括 系统指令 + 用户的问题 + 检索信息  =》 模型回答

这样就构成了知识有来源、信息有更新、逻辑有支撑的一个大模型问答系统。

## RAG 应用场景
可控的企业级问答系统，比如法律的问答系统， 
这个是不能出错的， 然后医疗的知识助手、客服系统， 都离不开rag。


rag 不仅是技术术语， 它代表的是大模型向真实世界靠拢的一个转折点。
让AI学会看数据、再引用证据、然后组织语言， 最后回答。


- 如果公司违法辞退了员工，具体的赔偿标准是怎么计算的？
- 如果员工严重违反公司规章制度，公司可以开除他吗？
