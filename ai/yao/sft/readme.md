# 模型微调SFT

AI 领域非常关键的技术， 模型微调SFT

Supervised Fine‑Tuning 监督微调

想像一个能和你聊哲学、去谈天说地、还能帮你去写代码的通才， 这就是LLM。
但它对财务的一些细节信息，或者法律的条款时，可能答得不在你的点上。
像挪威对阵英格兰， 索尔洛特没有给哈兰德传球，错失2：0 干掉英格兰的时机。

为什么？

其实很简单， 懂得多不等于懂得精。

怎么办？

像我们人一样，需要单向补课。对大模型也一样，也需要给大模型进行补课。这就是模型微调。

就是在大模型，经过通俗教育的基础上，我再去给它做一些专项的一个教育。

比如之前的pre-trained model, 它是一个通才。我们给它加一些领域的数据，然后
经过SFT， 那它可能就是一个专家了。

Qwen\GPE\DeepSeek 他们是经过了大量的数据做训练， 它是什么都懂的通才。

微调就是用更专业的数据， 去喂给他们， 让它从什么都懂，到我懂的很深。

比如在医疗、法律

那怎么去做微调呢？

微调有三板斧

- 做参数优化

预训练模型是面向大众的， 你只需要去微调它的可能 1% 或者到 10%的参数 就能够特化成像金融、政务、法律的一个专家模型。

比如LORA 技术， 我们只需要去训练很少量的参数， 就好像给模型，带了一幅专家的眼镜。

你可以把大模型的参数想象成 几万个旋钮。预训练完了，这些旋钮拧到了一个「什么都能答」的位置。微调就是
再拧其中一小部分旋钮，让它们偏向你的领域。                                                          
                                         
比如法律场景：原本旋钮在「通用回答」档位，微调就是把它往「法条逻辑、术语严谨」的方向拧一点。拧完以──
，模型再看到问题，激活的路径就变了——优先走法律相关的神经元，而不是去调取「怎么烤蛋糕」的知识。

不是重造脑子，是调偏重。

- 数据
你不能指望模型都通过维基百科、百度百科去学习医疗知识，我们用小而精且准确的数据去训练我这个模型。 

比如用1000条 标准的法律文书， 1000条金融分析师写的债务报表， 会比100万 或者 1000W 没有标注的， 这个行业的数据，会重要的多的多。

- 领域约束

从让模型天马行空的讲，到收敛到某个领域。大模型天性就是能说会道，但专业领域，其实
容不得那些看似有理的胡说八道。

所以我们在微调的时候， 可以加上逻辑约束。比如： 法律条款的前后的一致性。

让模型不只是说的像那么回事，而且是绝对是对的。


## 怎么去做微调呢？

- QLora
用小的显存去干大的事情， Q 指的就是量化， 相当于原始参数是32bit, 我们把它压缩到 4bit。
然后通过压缩的形式，让它减少显存的占用。 

第二个关键点就是LORA, 我们只去调关键层， 显存能够减少到90%， 一张4090 我们玩游戏的卡
也能去训练， 它的成本大概减少了 85%, 速度提升3-4倍。

如果没有显卡， 没有资源， 可以通过Prompt Engineering  PE

你让模型去回答一个法律的问题

你可以给它一个非常细致的提示词

比如 请根据民法典的第22条， 来回答这个问题。 

这样，通用大模型回答的精准度， 能够从 50% 可以提升到 85%。 

而你的一个参数都没动， 这个需要你懂法律， 或者你有特定行业的背景。

因为他需要让你去问出一个非常好的问题  提示词就是钥匙， 先通过这个钥匙， 把这个门打开，
然后再看我通过这个钥匙呢， 打开这个门是顺利还是不顺利，然后再去决定我是不是要通过微调
把我这个门锁给换掉。

微调界的万能钥匙。


为什么微调重要?
因为训练门槛越来月底， 但效果越来越好

企业部署的专家AI中，80%用的都是微调方案。
用1000条或1w条数据， 再加上单个设备，我就能够去训练一个很好的专家模型。

最后结合QLora技术， 我的总的一个训练成本，可能是我全参微调的10%

所以我觉得再AI的下半场， 比拼的不是谁的参数大， 而是谁通过微调， 
训的准， 训的快， 训的好。 

所以， Fine-Tue 不是一个可选项， 它是一个从大而范， 到精而准的必经之路。 

就好像一个再聪明的学生，他也需要各种专业的老师，通用大模型，也需要我们去精雕细琢。

这样能真正的落地。


- 和RAG 的区别

SFT‑LoRA 修改模型参数，把行业知识内化进模型；RAG 不改动模型，推理时调取外部文档。知识更新时微调要重训，RAG 仅更新知识库即可。

## 让 Gemma-2B 学会扮演甄嬛

Gemma-2B 是 Google 开源的轻量级大语言模型（20 亿参数），基于 Gemini                           同源技术，性能在同尺寸模型中领先，消费级 GPU（如 RTX 3090）即可运行。

- 数据集：huanhuan.json，内容是甄嬛风格的问答对
- System prompt："现在你要扮演皇帝身边的女人--甄嬛"
- 输入格式：system 指令 + 用户问题 + 期待模型用甄嬛的口吻回答
- 训练方式：LoRA（r=8，目标模块覆盖 q/k/v/o/gate/up/down 全连接层），3 个 epoch
- 最终效果：你问"你是谁？"，模型会用甄嬛的语气回答，而不是通用大模型的答案

一句话总结：就是用 LoRA 低成本微调，把 Gemma-2B 从「通用助手」变成「甄嬛角色扮演 bot」，属于
  风格/人格模仿 类的微调 demo。不是用来解决实际业务问题的，而是演示微调流程怎么跑通。
