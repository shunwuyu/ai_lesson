## 数据分类
- 训练集
  好像培养一个学生， 用的教材
- 验证集
  每次的课后试验， 帮助学生查漏补缺
- 测试集
  期中期末考试， 验证学生学得怎么样， 成绩好不好。

  训练集目的让模型去寻找数据的一个规律
  验证集就是频繁的去验证或者测试我的模型 训练好坏， 来帮助我们去调节一些参数。
  测试集就相当于我去验证我训练好的模型， 它在我没有看到的数据上的一个泛化能力。

  模型不是死记训练数据，能对全新没见过的陌生输入，做出正确、贴合逻辑的回答与判断的能力。

## 怎么划分
我有一批数据， 怎么划分训练集、验证集和测试集？
没有确定的划分标准， 但是有一些被验证过的套数。

如果数据量少的话， 6：2：2 的划分方式

但如果几百万条以上， 验证集和测试集 1W 条就好， 其余的都是训练集。
保证训练集数据量是足够大的，能覆盖很强的多样性。

当我们的模型结构就比较小， 参数量比较少的时候，把数据更多的留给训练集。
把验证集缩小， 总的目的提高模型的泛化能力， 而不是死记硬背数据集。

## 交叉验证
当我们数据量比较小的时候，如果是静态（6：2：2）划分，很容易去让模型直接偏科，交叉验证让
模型换个角度看数据。

三种方案

- 流出法 Holdout
  8:2
  7:2:1 
  这种方式很快速， 但是问题在于这是一个固定的划分， 很容易受划分的影响。
  单纯的 20% 测试数据可能就是 80%里没出现过的长尾数据， 没学习到， 那么
  llm对这些数据就是很差的一个学习和理解能力。
- 留一法 Leave One out
  每次留一分样本去做测试， 其他的都拿去训练， 好处是非常的精准（学得多）， 
  坏处是计算量大，适用于小的数据量集
- K折交叉验证 K-fold
  把数据集随机分成 K 份， 每次留一份去做测试， 其他的都拿去训练。
  k次循环过后综合去评估我的模型的性能。通常k=10 
  优势在于比单次的划分更稳定， 更能测试我模型的泛化能力
  完整走完 K 轮，每份都经历一次测试、K-1 次训练。

## 数据管理通路

采集-> 清洗 -> 标注 -> 管理

llm 时代， 是一个动态的数据管理流的系统， 有几个关键策略。

- 它会循环的去利用数据， 它会找到某些数据是比较差的，可能会影响模型性能的。
  进行重新的采样和清洗。
- 动态验证集选择
  他不是死的， 而是在大模型训练过程中， 动态去选择验证集
- 知识覆盖率分析
  大模型需要的数据是大量的， 不同的类型， 不同的语言等等，分布的测量，
  看是否在每个类型是不是分布均匀的，避免数据盲区。
- 数据去重，数据污染的检测
  重复数据会让模型过度记住同款内容，权重偏向重复样本，容易过拟合，还白白增加训练算力与时长。
  精确去重（完全一模一样）、模糊语义去重
  数据污染检测 
  脏数据污染：夹杂违规内容、错误谣言、乱码、低质量爬虫垃圾文本，检测过滤这类劣质内容也属于广义污染检测。

  llm 时代， 数据管理的重要性和不可或缺性。

  数据也不是越多越好，数据划分是一门艺术。

  在模型之上， 是数据的艺术， 模型之下呢， 是数据的地基。

  如何科学的管理数据， 动态的组织数据，充分的利用好每个数据，才是真正能训练好模型的底层逻辑。
