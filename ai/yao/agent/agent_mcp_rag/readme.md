# Agent MCP RAG

- Agent 
AI Agent 是能自主理解目标、规划步骤、调用工具完成任务，可自主闭环执行复杂工作的智能 AI 智能体。
Claude Code/Codex 编程智能体
阿里悟空 钉钉出品的企业全域办公智能体平台
面向 C 端大众的通用全能型 AI Agent

- RAG

RAG 即检索增强生成，会先从自有知识库调取真实资料，再结合大模型生成答案，能有效解决模型幻觉、知识老旧的问题。
知识库、Cursor 依托 RAG 全局索引本地代码仓库

- MCP 
MCP（模型上下文协议）是 Anthropic 推出的 AI 通用开放标准，如同 AI 领域的 USB-C 接口，可让大模型统一、便捷地对接各类外部工具、数据库与服务数据源。

高德地图 MCP 自动生成旅游路线、通勤方案
GitHub MCP AI 直接拉取仓库代码、审阅代码，自然语言完成 Git 全流程操作。

三者合体， AI就成人工智障变成超级大脑。

## RAG 
让AI变成一个超级学霸 ， 以前AI回答问题全靠背书（预训练），训练的时候，见过什么就去说什么，没见过就胡编乱造。
但RAG 不一样， 它会先去查字典，然后从知识库里找相关信息。然后基于这些真实的资料去生成回答。
RAG 让Ai告别了张口就来的时代。

## Agent

AI 的超级助理， 你跟普通的AI说帮我筹备新品的发布会， 他可能会回一堆。 但Agent 不一样， 它会像一个私人助理一样，先去分析任务， 看看发布会需要哪些环节， 然后去指定计划， 比如像场地、媒体、物料， 像流程， 接下来，就是主动执行了， 去调用各种的工具， 去完成每个子任务， 关键它还有自主性， 不用你一步一步的去指导，它能够独立的思考和行动。

## MCP
AI 万能转接头 
企业的数据库、以及各种的在线工具、天气、股票、新闻等。我们定义为tools和资源。

## 三者关系
三个从来不是独立存在的， 他们是一个完整的AI系统的一个核心能力。

RAG 给Agent去提供大脑， Agent做决策需要准确的信息， RAG 就是它的一个知识来源。
MCP去给Agent提供手脚， Agent 想去执行任务， 需要去调用外部工具， MCP就是它的一个接口标准。
MCP也会给RAG 提供眼睛， RAG 要去检索知识， 需要去链接各种的数据源， MCP让这个过程标准化了。 

举例：

帮我分析一下 今年的投资机会
RAG 先上场， 从财经数据库检索最新的市场信息、政策文件、行业报告。然后Agent开始工作， 分析你的风险的偏好， 指定投资的策略，规划具体的步骤。MCP去提供支撑，去调用股票的API, 去获取实时的数据，链接银行系统去查看账户，对接分析工具生成图表。

单独的RAG, 它只会去查资料不会思考。
单独的Agent 会思考， 但可能会基于错误信息去执行。
没有MCP, Agent想法很好， 但执行不了。
三者结合， 就变成了有知识、会思考能够去执行。

现在AI领域都在讲Agentic RAG, 都在推MCP 标准。
因为未来的AI,不是单一的能力， 而是这种复合型的系统。

想像一下你的AI助手， 既有图书馆一般的知识储备， 又有策略大师的思维能力， 就像瑞士军刀一样去调用，去调用各种工具， 这就是RAG+MCP+Agent 的终极形态。
