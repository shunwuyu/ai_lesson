# Workflow 和 Agent 

## Workflow

昨天的langchain 代码

本质上是一套预定义的流程，可以把他想像成一套生产的流水线，每一步都有明确的输入和输出。 每个节点的逻辑是固定的，当条件满足，就会进入下一环。在技术上， 它通常变现成一个编排好的工作流引擎， 一个由AI调用，条件判断，循环逻辑组成的一个图谱。

const creativeChain = storyPrompt.pipe(creativeModel).pipe(outputParser);

该代码定义一条串行 AI 工作流：以 storyPrompt 作为起始输入源，依次流经 creativeModel 生成创意内容，再交由 outputParser 解析输出，形成固定顺序流转的链式执行链路。

LangChain 是用于快速搭建 AI 工作流的开发框架，灵活构建各类智能任务链路。

## 举例

你让workflow 处理一份简历， 先解析pdf, 第二部提取工作经历，第三步匹配岗位需求，最后输出一个推荐度。
整个过程是确定性的， 你完全知道它会怎么去走。这就是workflow的优势。可控、稳定、可复现。但它也有天然的局限，他不会质疑自己被安排的步骤，更不会，在遇到意外情况的时候，自动创照新的路径。


## 思考

AI 工作流也能干活啊， 它和Agent 有什么区别？ 
很多人会混淆， 反正都是AI 在执行任务。 

但是当我们真正深入到技术和应用层面， 他们的区别， 几乎决定了AI 未来的一个方向。

## Agent

和workflow 逻辑完全不一样， 他不是走在一条预设的轨道上，而是站在一个开放的空间里， 自己去探索怎么去做。从技术角度， 一个Agent往往具备三个大的核心能力。

第一个，感知环境。
通过输入文本、图像、数据来理解任务。
第二个，规划路径。
不是预先定义好，而是动态去生成任务链。
第三个，执行行动
去调用工具完成子任务，并不断的去调整，

举个例子，你让Agent 去规划旅行，他可能会先去搜一下机票和酒店价格。再去查天气和签证要求。然后结合你的偏好和预算，动态去权衡方案。整个过程， 他可能走的路径和你预期的完全不同，这就是Agent的关键区别，自主性和适应性。
他不会被一个固定的流程限制，而是像一个思考中的个体。所以，他们本质的一个区别在于: Workflow 是确定性的执行， Agent是一个不确定的探索。 workflow就好像一条高速公路，路线很清晰方向固定，效率很高，但只能到达既定的终点。

agent更像一个司机，他知道要去哪，但可以随时改道，尝试新路。甚至在目的地发生变化的时候，自行调整。

所以，从产业角度， workflow 更适合流程化，重复性的任务。
比如审批流程， 数据清洗， 客服的SOP

Agent 更适合动态的一个角色场景。比如：复杂搜索，策略规划，跨  领域问题的求解。

从技术栈的领域， workflow 更依赖于规则， 图形化编排，流程引擎。
Agent 更依赖于大模型的推理能力和记忆机制，包括它的工具调用。

从未来角度看， 他们两会不会合并？
答案是肯定的，未来的系统往往是两个的结合，底层是workflow去提供稳定性和可控性。上层是Agent 提供灵活性和智能型。
workflow是Agent的骨架， Agent是Workflow 的大脑。

如果只有workflow, 系统就好像一个没有创造力的一个工人。但如果也只有Agent呢？ 他可能很充满， 但是不可控， 不可预测。 两者结合起来， 既高效率又有智慧。
