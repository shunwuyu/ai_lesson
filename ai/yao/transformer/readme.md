  让chatgpt 真正走进了大众视野， 也支撑起了整个的大语言模型。

让我们学习一下Transformer基础原理，包括它怎么去处理语言，解决序列建模，甚至去推导一下公式， 真正理解它。

语言是一个序列（sequence）
所谓序列， 每一个词的意义， 都可能依赖上下文。
比如这句话 “我以为今天要放假了”， 老师说 “别做梦了”。
这两句话中， 出现了长距离的语义依赖（梦， 今天放假， 第二句话虽然省略了主语，它指向了第一句话的主语“我”）和情绪转折（极度期待/乐观”到“瞬间清醒/失落）。
需要模型去记住“放假”这个设想， 以及“别做梦了”这个否定态度。
早期的模型里面， 最常见去处理序列问题的，主要是两大类：
- RNN 循环神经网络 Recurrent Neural Network
RNN 擅长处理有先后顺序的数据，
比如语音转文字，一句话的字音是按顺序来的，RNN 顺着挨个听，结合前面的发音判断当下的字，比如听到 “面”，能结合前面的 “番茄鸡蛋”，准确识别，靠顺序记忆搞定时序内容。

RNN 就是循环神经网络，处理文字时按顺序挨个读取字词，做完当前词计算才轮到下一个，能记住一点前文内容，但只能串行运行，处理长文本速度慢，很容易丢失前面的细节信息。

- LSTM 长短期记忆 Long Short Term Memory

LSTM 算是改良版 RNN，专门解决原版容易忘前文的毛病，自带记忆闸门，能选择性留存久远信息、丢掉无用内容，依旧按顺序处理文本时序数据，长句子记忆效果强很多。

比如一句话：“我昨天买了一只橘猫，它特别调皮，今天总趴在窗台晒太阳”
普通 RNN 读到末尾晒太阳时，早就忘了开头买的是橘猫；
LSTM 靠闸门牢牢留住 “橘猫” 这个关键信息，后面提问是什么宠物，能准确答出来。

他们天生适合处理语言序列， 但有三个问题：
1. 信息遗忘
序列在逐步处理的时候， 早期的概念可能会随着时间的推移而慢慢衰减。
比如前100个词的某个细节， 到第200个词的时候，模型可能就完全忘记了。
举例
开头你明确要求：所有接口返回统一格式 {code,msg,data}，全部用 JS 原生语法（前文前置规则）
后面让模型连续写十多个接口代码，写到后半段，它慢慢忘了开头的格式约定，偷偷自定义返回结构，甚至用上了第三方库，就是典型的上下文早期细节随序列变长慢慢丢失。

本质就是上下文窗口里早期 token 权重变弱，注意力没法牢牢盯住开头信息。
每堂课 45分钟， 下课尿尿 
- 并行效率低
传统的RNN模型，只能一个词一个词的去处理， 当前词没有处理完， 不能去处理下一个词。导致， 在训练和推理任务中， 速度非常慢。尤其在长序列任务里面，问题格外明显。
就像排队过独木桥，RNN 就是单人单行道，必须前一个人完全走完，下一个才能动。
例子：
你要处理一句话：【今天想吃番茄鸡蛋面】
RNN 流程：先算「今」→算完才能算「天」→完事再算「想」，挨个排队算，一个卡死后面全等着。句子越长，排队队伍越长，耗时成倍拉慢。
反观 Transformer 相当于把整句话所有字同时拿出来一起运算，效率天差地别。

- 长距离的依赖弱
LSTM引入了记忆机制， 但它在建立远距离词的关系的时候，任然很弱。
比如要让模型理解前面是原因，后面是结果， 长长它只能去捕捉局部的上下文，很难去捕捉全局的上下文。

transformer 的提出和出现是为了解决这三个问题。 

1. 它用self-attention 去替代循环的结构。

自注意力（self attention）能让文本所有字词互相计算关联、并行运算，解决 RNN 串行慢、易忘前文的问题，出自谷歌论文《Attention Is All You Need》，一作 Ashish Vaswani，联合作者 Noam Shazeer 创办 离开google, 入职 OpenAI，担任架构研究负责人。

让当前词， 能够去看到所有的其他词，不再受距离的限制。

就拿句子：小猫偷吃了桌上的鱼，它吃得很香
RNN 顺着读，读到最后的 “它”，隔了一串词，容易忘了指代谁。
Self-Attention 可以让 “它” 直接一眼定位开头的小猫，不受单词间距远近限制，全局所有字词互相关联，并行一次性完成计算，速度远快于串行的 RNN。
这样会不会有些计算浪费？ 比如香和桌没啥关系啊？
确实会有无效计算，但自注意力自带权重打分，自动弱化无关词，兼顾全局视野与效率，简单举例：
句子：小猫偷吃了桌上的鱼，它吃得很香
计算时所有字词两两全都算一遍关联分数，“香” 和 “桌” 确实会产生计算，看似浪费；
模型算出权重：香↔吃、香↔小猫分数极高，香↔桌子权重极低，近乎忽略；
2. 完全基于矩阵运算
能够去高度的并行， 支持GPU的大规模并行训练。
文本->分拆->词元->嵌入->矩阵 

Transformer 一次性把整套矩阵丢进 GPU，所有单词的计算同时同步跑完，相当于几十上百个洗碗工人同时开工，GPU 多核算力全部拉满，长文本训练速度碾压串行的 RNN。

3. 通过多层的堆叠，去捕捉复杂的语义信息，实现对长文档的全局建模能力。

小明踢翻了奶奶放在院子的花盆，他赶紧弯腰把碎片收拾干净
第一层（底层）：只认单个词语，分清谁是小明、奶奶、花盆，只认识单词本身；
第二层：看懂短句关系，明白是小明踢翻花盆，花盆属于奶奶；
高层堆叠层：全局看懂全文，精准知道后半句的「他」指代小明，理解整件事的前因后果。
一层一层叠加处理，浅层抓字词，中层抓短句，深层吃透整篇逻辑，堆叠越多，越能搞定长篇长文的复杂语义，这就是 Transformer 多层堆叠。

## transformer 架构
可以看成是由想同的多个层去堆叠而成的。
每个模块包含三个大的核心组件。
1. 词向量嵌入（embedding）和位置编码（position encoding）
转 Token ID：文字拆成一个个 token，对应字典里唯一的数字编号（比如 “猫”=128），这串数字只是序号，没有语义；
查表 Embedding：拿着 ID 去嵌入权重表（模型预训练全程一起训练出来的），取出对应的一串小数数组（词向量）；
最终把所有 token 的向量拼成矩阵，数字自带语义远近关系，才能送入自注意力做矩阵计算。
可以这么理解：Token ID 就是字典的页码索引，只用来定位；嵌入权重表才是完整字典，索引查到页码，就能取出带语义的词向量内容。

因为embedding不认文字顺序，位置编码就是额外加上一串数字，告诉模型每个词在句子里的先后位置。

拿两句话举例：
猫追狗
狗追猫
单词完全一样，只是顺序不同。
只靠词嵌入，模型看不出区别。把位置编码（正弦函数或预选函数得到）加到向量里，标注每个词的位次，模型就能分清谁在前、谁在后，精准区分两句话完全相反的含义。

公式表达 X=E+P

第二个组件叫 self-Attention,它是transformer 最核心最灵魂的模块。
每个词都是通过 Q K V 来进行信息的交换。每个词不能孤立理解，必须结合上下文才能懂意思，QKV 就是用来计算词与词之间的关联权重，完成上下文信息交换。

Q Query 我需要找那些词的信息 

小红买了蛋糕，她很开心
拿 token「她」当作当前词，生成自己的Q（Query 查询）
它的诉求就是：我要在整句话里，找到能解释 “她” 是谁的词语。
句子里其余词语（小红、买、蛋糕、开心）各自拿出 K (Key 特征)，和「她」的 Q 逐一比对相似度。
Q 和「小红」的 K 匹配度最高，权重最大，后续就会大量提取「小红」对应的 V 数值。

K Key 特征  代表其他词的特征
  小红的 K：特征是人物、女性人名
  买的 K：特征是动作动词
  蛋糕的 K：特征是食物、物品
  她的 K：特征是第三人称女性代词
  开心的 K：特征是情绪感受
V Value 信息 存着这个词完整的内容信息，是最后要拿来取用的本体数据
K 是用来比对的身份证，V 是此人的全部档案，二者是两套资料，不是包含关系。

Multi-Head Attention

咱们接着用「小红买了蛋糕，她很开心」来讲多头注意力，大白话讲透：
单头注意力（刚才讲的）
就好比一个观察员，只盯着一件事分析：
拿着「她」的 Q，只筛选性别身份特征，精准锁定小红，只搞定指代是谁这一个问题。
优点简单，缺点看得太片面，只能捕捉一种关系。
多头注意力：拆成好几个独立观察员（多头）同步干活
把 QKV 各自分成好几组，每一组就是一个头，每个头负责看一种不同的关系，分头计算，最后把所有头的结果合并起来。
拿咱们的句子举例，拆 3 个头好理解：
头 1（身份头）：专门看人称指代
只比对人物性别特征，和单头一样，判断「她 = 小红」。
头 2（动作关系头）：专门看主谓动作
抓取「小红」和「买」的绑定关系，明白是小红去买蛋糕，理清动作主体。
头 3（情绪因果头）：专门看因果情绪
关联「买蛋糕」和「开心」，读懂：买到蛋糕，所以小红心情愉悦。
通俗类比
单头注意力 = 一个人只用眼睛看东西，只能看清外形；
多头注意力 = 同时用上眼睛、耳朵、大脑思考，多维度同步观察。
多个头互不干扰，各自抓取一类语义关系，有的看指代、有的看动作、有的看情绪、有的看修饰。

多头相比单头，捕捉关系更多维、理解语义更全面、泛化能力更强。
- FFN X = 我的激活函数
全称 Feed Forward Network 前馈网络，它紧跟在多头注意力后面

多头注意力负责词和词之间横向的上下文关联，FFN 只管单个词自身纵向的特征升级，每个 token 都是独立运算，互不干扰。

结合咱们例句举例：小红买了蛋糕，她很开心
注意力已经搞定「她 = 小红」，轮到 FFN 单独加工每一个词：
单独优化「小红」的特征、单独优化「蛋糕」、单独优化「她」，各自升级自身信息。
比如强化「小红是人」「蛋糕是甜品」「开心是正面情绪」这些内在属性，让特征更饱满。

文本先做词嵌入加位置编码，送入多头注意力抓取上下文关联，再经 FFN 强化单词语义特征，配合残差归一化，层层堆叠后输出最终结果。
