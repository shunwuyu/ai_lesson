# 数据库连接和模型

你要查数据库里的用户列表，得手写一长串 SQL 字符串：
```
SELECT id,username,email FROM user WHERE page=1 LIMIT 10;
```
新增用户还要拼INSERT、改数据拼UPDATE、删数据拼DELETE。

手写原生 SQL效率低，容易出错，有一定的门槛，也不方便维护。

通常会使用 ORM（Object Relationship Mapping）框架来操作数据库。

把数据库的表，直接变成 Python 里的类；把表里的一行数据，变成类的实例对象。

SQLAlchemy 是功能最强大，且社区最活跃的 ORM 库，并且还支持异步， 可以在 FastAPI 中使用。

## 为什么需要 ORM？
在前面两个阶段，我们一直用 Python 的 dict 或 list 来"模拟"数据库：
```
# 阶段 1-2 的"数据库"——进程重启就没了
fake_db = {
    1: {"name": "张三", "email": "zhang@example.com"},
    2: {"name": "李四", "email": "li@example.com"},
}
```
这种方式的问题：
- 数据不持久	服务重启，所有数据丢失
- 无并发安全	多个请求同时写，数据会错乱
- 查询能力弱	无法做复杂筛选、排序、联表
- 无类型约束	字典可以随便塞，没有任何校验
- 无迁移机制	表结构变了，旧数据没法自动迁移

ORM（Object-Relational Mapping） 解决了这些问题——它让你用 Python 类来操作数据库，而不用直接写 SQL 字符串。

你写的代码                         ORM 做的事                         数据库
─────────────────────────────────────────────────────────────────────────────
user.name = "王五"    →    UPDATE users SET name='王五'   →   MySQL
db.add(user)                         WHERE id = 3;               SQLite
db.commit()                                                        PostgreSQL


SQLAlchemy 是 Python 生态中最强大、最成熟的 ORM 框架。FastAPI 官方文档推荐使用它。
