- 分割最简单的就是按照字符，比如换行符 \n
但并不是每一行一个 Document，而是要设置一个 chunk size，按照换行符分割好的内容加入到这个 Chunk，当达到 chunk size 后，再继续生成下个 Chunk。

这个 Chunk 也是 Document 对象，只是文档内容是分割好的一个个大小合适的块。

const textSplitter = new RecursiveCharacterTextSplitter({
  chunkSize: 500,    // 每个分块的字符数
  chunkOverlap: 50,  // 分块之间的重叠字符数
  separators: ["。", "！", "？"],  // 分割符，优先使用段落分隔
});

完整切割执行顺序（递归逻辑）
第 1 步：用最高优先级分隔符「。」全文拆分
把完整文本按 。 切成一段段短句；
遍历每一段，判断字符长度是否 ≤ 500：
长度≤500：直接保留为候选块；
长度＞500：进入下一级分割。
第 2 步：超长片段，用次一级分隔符「！」再切
对上面超过 500 字符、无法用句号拆分的文本，改用 ！ 分割：
按 ！ 拆分文本；
再次校验每段长度：
≤500：保留；
仍＞500：进入下一级。
第 3 步：还超长，用最低优先级「？」分割
对前两步切完依旧超长的文本，用 ？ 分割，再校验长度。
第 4 步：所有分隔符用完，文本还是超过 500 字符
递归分割符全部遍历完，片段依然大于 chunkSize，强制硬切：无视语义，直接截断成 500 字符一块。
第 5 步：生成分块 + 处理 chunkOverlap 重叠
收集所有符合长度的小片段，拼接组装成最大不超过 500 字符的完整块；
相邻两个块之间，向后重叠 50 个字符：
块 1 末尾 50 字符，会完整出现在块 2 开头；
作用：避免语义被切断、上下文断层。
