## 阶段2：视觉和交互优化

现在已经初步完成了项目骨架。这一轮我们对视觉和交互进行优化。## 1. 漂浮岛当前网格只有薄薄的"草皮"，下面什么都没有，看起来像悬浮在空气里。我想要它像"挖出来的一块土地，下面有厚度"——板下面要有"土块"层次，能看出"这是一块完整的、被托起来的小世界"。不需要写实地质，几层逐渐变小、颜色变深的方块叠在一起就够。## 2. 移轴模糊（tilt-shift）画面上下要各有一条"模糊带"，让中间清晰、上下糊掉，模仿移轴镜头拍小模型的感觉。- **关键**：用 CSS 实现（`backdrop-filter: blur` + `mask-image`），不要用 Three.js 后期处理- 模糊带的高度上下不一样：上面薄一点，下面厚一点- 模糊带边缘要软过渡，不要硬边## 3. 暗角（vignette）画面四角微微变暗，让眼睛聚焦中心。CSS radial-gradient 实现。**颜色不要偏暖偏黄**，用偏冷的暗灰，否则和奶油背景叠出来会"发脏"。## 4. 物体细节升级当前的树、房子、瓦片都太简单，要让它们更有"积木玩具"的细节：- 草地上散几丛小草簇- 水塘做成"上层半透明水面 + 下层深色水底"两层- 石头不要光滑方块，叠几个高低不齐的小碎块- 树要"主大冠 + 几个偏移小冠"的体积感- 房子要有：身体、踢脚线、屋顶（四角形锥）、屋檐、门、两侧发光小窗、一个烟囱## 5. 形态稳定同一个格子刷新后，物体的随机姿态（树的旋转、草丛位置等）应该不变——也就是说要用"基于格子坐标的种子"生成随机数（比如 mulberry32 + 哈希），不是 Math.random。这样切换存档时画面不会跳变。## 6. 路自动连当前的土路每块都长得一样。希望相邻的路自动连成"直/弯/T/十字"——放下一格路时，根据它四个邻居哪些是路，决定它的形态。**关键**：setCell 之后要遍历四个邻居中是 path 的格子，全部重新渲染，让它们的连接形态及时更新。## 7. 放置动画点击放置时，新东西从天上掉下来落入位置，~0.3 秒，要有一点点弹性回弹的感觉，不要纯线性。

## 细节补充

现在画面已经有"摆在桌上的小模型"的感觉了。这一轮加几个会动的小细节让它"活"起来。## 1. 飘云 + 云影天空上方有几朵云慢慢飘过，地面要能看到云影掠过的样子。- 云用方块化的几何体堆叠（dodecahedron 之类），不要圆滑球体——保持"积木"风- 数量大概 8-10 朵，集中在板子上方一定范围内（不要散到画面外）- 高度不要太高，差不多板子尺寸的一半左右——太高视觉占比就小了- **关键**：云影不要单独画一层 plane，靠 Three.js 的阴影系统自动产生（每朵云的 mesh 设 castShadow=true，地面已经 receiveShadow）- 云缓慢往一个固定方向飘，飘出范围就 wrap 回另一边，循环## 2. 烟囱烟雾每栋房子的烟囱顶持续冒烟。- 隔一段时间发射一颗小灰白球（不要太密，3 秒里冒出 5-8 颗的密度差不多）- 烟球向上慢飘 + 横向轻微随机抖动- 透明度从有到无、尺寸从小到大，寿命到了就消失并 dispose 它的材质- 注意房子有旋转，发射时要把烟囱顶位置正确转到世界坐标- 总数有上限（避免 60 颗以上同时存在），到了就不发新的## 3. 悬停高亮升级当前鼠标悬停的格子有一个金色描边，让它"呼吸"——透明度像呼吸一样缓慢起伏。- 频率要慢，给人"温柔脉动"的感觉，不是"急促闪烁"- 颜色保持金黄## 完成标准打开网页应该能看到：- 云缓慢飘过 + 地面有柔和的影子掠过- 每栋房子持续冒烟（细烟柱缓缓升起）- 鼠标悬停格子有温柔脉动的金色边框如果云影看不见，先检查云的 castShadow 和 sun 的 shadow.camera 范围——但**不要**给云单独加一层 plane 装阴影。
